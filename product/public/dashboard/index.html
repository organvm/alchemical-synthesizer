Agent & developer surfaces

    
The same capability core, four ways. Authenticate with Authorization: Bearer <your key>.

    

      
REST API

      
curl -s $ORIGIN/api/v1/modules | jq
curl -s -X POST $ORIGIN/api/v1/render \
  -H "Authorization: Bearer $KEY" -H "content-type: application/json" \
  -d '{"module":"Azoth"}'

      
MCP (Model Context Protocol)

      
curl -s -X POST $ORIGIN/mcp -H "content-type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"tools/list"}'
# add Authorization: Bearer $KEY for costed tools (render_specimen)

      
ACP — Agent Communication Protocol

      
curl -s -X POST $ORIGIN/acp/runs -H "content-type: application/json" \
  -d '{"agent_name":"brahma","input":[{"parts":[{"content":"render from Nebula"}]}]}'

      
ACP — Agent Client Protocol

      
curl -s -X POST $ORIGIN/acp/jsonrpc -H "content-type: application/json" \
  -d '{"jsonrpc":"2.0","id":1,"method":"session/prompt","params":{"prompt":"list modules"}}'
# native stdio transport:  node bin/acp-stdio.js
